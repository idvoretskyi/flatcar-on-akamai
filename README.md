# flatcar-on-akamai

Zero-touch [Flatcar Container Linux](https://www.flatcar.org/) on
[Akamai Cloud (Linode)](https://www.linode.com/), provisioned with
[OpenTofu](https://opentofu.org/) and configured with
[Ignition](https://coreos.github.io/ignition/) generated by
[knuckle](https://github.com/projectbluefin/knuckle).

The deploy is **seamless**: one-time image upload, then `tofu apply` brings up a
fully-configured Flatcar node with your SSH key already installed — no rescue
mode, no serial-console babysitting, no `flatcar-install` dance.

> **Status:** deployed and verified on Akamai Cloud (Linode, `us-ord`).
> Flatcar Container Linux 4593.2.2 booted zero-touch, Ignition applied on first
> boot, `ssh core@<ip>` worked immediately.

## How it works

```
 one-time          per-deploy
┌────────────┐    ┌──────────────────────────────────────────────┐
│ make image │    │ make ignition  ──▶  build/ignition.json       │
│  uploads   │    │   (knuckle --emit-ignition | butane)          │
│  Flatcar   │    │                                               │
│  image as  │    │ make apply (OpenTofu):                        │
│ private/id │    │   linode_instance       metadata.user_data ◀──┘
└────────────┘    │   linode_instance_disk  ◀ from private/id
                  │   linode_instance_config  kernel=direct-disk
                  │        │ boot
                  │        ▼
                  │   Afterburn reads user_data ▶ Ignition applies
                  │        ▼
                  │   ssh core@<ip>
                  └──────────────────────────────────────────────┘
```

Why this shape:

- **Official Flatcar Akamai image** (`flatcar_production_akamai_image`) ships
  Afterburn, which reads the Linode **Metadata service**. We hand Ignition in
  via `metadata.user_data` — no in-VM installer needed.
- **knuckle generates the Ignition.** This repo uses a small fork that adds
  `knuckle --emit-ignition` (generate the config, never touch a disk). knuckle's
  headless config schema is the single source of truth; a plain `butane`
  fallback exists for hosts without Go.
- **OpenTofu does the rest**: boot the image disk directly via
  `linode/direct-disk` with Linode's distro/network helpers disabled so Flatcar
  owns the system.

## Quick start

```bash
# 0. tools: tofu, linode-cli, jq, git (+ Go for the knuckle backend)
make tooling

# 1. configure
cp .env.example .env && $EDITOR .env      # set REGION, SSH_AUTHORIZED_KEY, ...
export LINODE_TOKEN=...                    # Linodes RW + Images RW

# 2. one-time image upload  (prints private/<id>)
make image                                 # then set IMAGE_ID in .env + tofu/terraform.tfvars

# 3. render Ignition
make ignition                              # BACKEND=butane to skip Go

# 4. deploy
cp tofu/terraform.tfvars.example tofu/terraform.tfvars   # set image_id + region
make init plan apply
make ssh
```

## Two ways to use knuckle

| Path | What | When |
| ---- | ---- | ---- |
| **A. Ignition generator (default)** | `knuckle --emit-ignition` produces the Ignition that OpenTofu delivers via metadata. Zero-touch. | Normal deploys. |
| **B. Native installer** | Boot knuckle, run `knuckle --headless --config` to `flatcar-install` onto a disk from Rescue Mode. | Documented alternative — see [docs/05](docs/05-knuckle-rescue-mode.md). |

The `--emit-ignition` flag lives in the fork
[`idvoretskyi/knuckle@emit-ignition`](https://github.com/idvoretskyi/knuckle/tree/emit-ignition);
see [knuckle/README.md](knuckle/README.md).

## Important caveats

- **SSH keys must be in Ignition.** Afterburn on Akamai cannot consume Linode's
  managed `ssh_keys` or the disk `authorized_keys` field. This repo puts your
  key in the Ignition config; the disk `root_pass` is a random throwaway.
- **Image upload limit is 6 GiB** and incurs a small storage charge.
- **Private networking** needs an Ignition unit using `COREOS_AKAMAI_PRIVATE_IPV4_0`
  — see [docs/06](docs/06-troubleshooting.md).
- **No spend until you choose it.** Offline validation (`make validate`) never
  contacts Linode.
